# Firmware

Two edge node implementations — same business logic, different hardware platforms.

| | ESP32-CAM (`esp32_sensor_node/`) | Raspberry Pi (`rpi_sensor_node/`) |
|---|---|---|
| **Language** | C++ / Arduino | Python 3.10+ |
| **Camera** | OV2640 via `esp_camera` | picamera2 (RPi Camera Module) or OpenCV (USB webcam) |
| **Sensor** | DHT22 + LDR via 12-bit ADC | DHT22 via adafruit-dht + LDR via MCP3008 SPI ADC |
| **GPIO** | `digitalWrite / analogRead` | `RPi.GPIO` |
| **MQTT** | `PubSubClient` + `WiFiClientSecure` | `paho-mqtt` |
| **Firebase** | `Firebase_ESP_Client` | Firebase REST API via `requests` |
| **TFLite** | TFLite Micro (8 KB tensor arena) | `tflite-runtime` (standard) |
| **Config storage** | SPIFFS (`/config.json`) | Filesystem (`data/config.json`) |
| **Template storage** | SPIFFS (`/bio/*.bin`) | Filesystem (`bio/*.npy`) |
| **FSM states** | Same 10 states | Same 10 states |
| **Anomaly scoring** | Same formula | Same formula |
| **Firebase paths** | Same | Same |
| **MQTT topics** | Same | Same |

## ESP32-CAM Node

### File Map

| File | Role |
|---|---|
| `esp32_sensor_node.ino` | Entry point; FSM orchestrator |
| `IrisCamera.h/.cpp` | OV2640 capture + 64-element feature extraction |
| `BiometricManager.h/.cpp` | SPIFFS template storage, enroll/match/remove |
| `AnomalyDetector.h/.cpp` | Sliding-window composite anomaly scoring |
| `AlertManager.h/.cpp` | Throttled alert dispatch (AWS MQTT + Firebase) |
| `StateManager.h/.cpp` | Finite state machine (10 states, legal transitions) |
| `ConfigManager.h/.cpp` | JSON config from SPIFFS + compile-time fallbacks |
| `SensorManager.h/.cpp` | DHT22 + LDR reads with EMA smoothing & retry |
| `ActuatorController.h/.cpp` | Relay + LED patterns + override expiry |
| `FirebaseManager.h/.cpp` | Firebase RTDB push/pull + offline ring-buffer queue |
| `AWSIoTManager.h/.cpp` | MQTT/TLS publish, subscribe, agent ACK handling |
| `MLInference.h/.cpp` | TFLite Micro ambient env risk wrapper |
| `aws_certificates.h` | X.509 certs (paste from AWS console) |
| `tinyml_model.h` | Embedded model byte array (generated by `models/model_converter.py`) |

### Required Arduino Libraries

Install via Arduino IDE → Sketch → Include Library → Manage Libraries:

| Library | Author |
|---|---|
| DHT sensor library | Adafruit |
| Adafruit Unified Sensor | Adafruit |
| Firebase ESP Client | mobizt |
| ArduinoJson ≥ 6.x | Benoit Blanchon |

ESP32 camera driver is bundled with ESP32 Arduino core ≥ 2.0.11 — no separate install needed.

### Setup Commands

```bash
# 1. Train ambient model → tinyml_model.h
cd models
pip install -r requirements.txt
python mock_data_generator.py
python model_training.py --epochs 100
python model_converter.py
# → writes firmware/esp32_sensor_node/tinyml_model.h

# 2. Edit credentials
#    firmware/esp32_sensor_node/data/config.json

# 3. Paste AWS X.509 certs into
#    firmware/esp32_sensor_node/aws_certificates.h

# 4. Arduino IDE
#    Tools → Board → AI Thinker ESP32-CAM
#    Tools → Partition Scheme → Huge APP
#    Tools → ESP32 Sketch Data Upload     (uploads config.json)
#    Sketch → Upload                      (flashes firmware)

# 5. Serial Monitor → 115200 baud to watch FSM transitions
```

### Wiring

```
ESP32-CAM GPIO 15 → Auth button (active LOW, pull-up to 3.3V)
ESP32-CAM GPIO 4  → DHT22 Data (10kΩ pull-up to 3.3V)
ESP32-CAM GPIO 34 → LDR (voltage divider to GND)
ESP32-CAM GPIO 26 → Relay IN
ESP32-CAM GPIO 2  → Status LED (or onboard LED)
3.3V / GND        → DHT22, LDR pull-up resistor
```

### Mock Mode (no hardware)

Add `#define SENSOR_MOCK` at the top of `SensorManager.cpp` to enable simulated
sensor data — useful for bench-testing the Firebase/ML pipeline without physical hardware.

| File | Role |
|---|---|

### Setup Commands

```bash
# 1. On the Raspberry Pi
cd firmware/rpi_sensor_node
pip install -r requirements.txt

# 2. Build and copy the TFLite model
cd ../../models
pip install -r requirements.txt
python mock_data_generator.py
python model_training.py --epochs 100
cp model.tflite ../firmware/rpi_sensor_node/data/ambient_model.tflite
cd ../firmware/rpi_sensor_node

# 3. Copy AWS certificates into data/
#    data/root-ca.crt   ← Amazon root CA
#    data/device.crt    ← device certificate
#    data/device.key    ← private key

# 4. Edit data/config.json with your credentials

# 5. Enable hardware interfaces
sudo raspi-config
# → Interface Options → Camera → Enable
# → Interface Options → SPI → Enable
sudo reboot

# 6. Run
python main.py

# Demo mode — no hardware required
SENSOR_MOCK=1 python main.py

# Run as a systemd service (auto-start on boot)
sudo cp iris-node.service /etc/systemd/system/
sudo systemctl enable iris-node
sudo systemctl start iris-node
```

### Wiring

```
RPi GPIO 4  (BCM) → DHT22 Data (10kΩ pull-up to 3.3V)
RPi GPIO 17 (BCM) → Relay IN
RPi GPIO 27 (BCM) → Status LED
RPi GPIO 22 (BCM) → Auth button (active LOW, internal pull-up)
RPi SPI0 (MOSI/MISO/SCLK/CE0) → MCP3008 SPI ADC
MCP3008 CH0 → LDR voltage divider
RPi CSI connector  → Raspberry Pi Camera Module v2/v3
```

> **Why MCP3008?** The Raspberry Pi has no onboard analog-to-digital converter.
> The MCP3008 is an 8-channel 10-bit SPI ADC (~$2). Alternative: replace the LDR
> with a BH1750 digital ambient light sensor (I2C, no ADC chip needed).

### Adding a Second Device

1. Duplicate `data/config.json`, change `deviceId` → `rpi_node_02`
2. Create a new AWS IoT Thing, download its certificates into `data/`
3. Run `main.py` — the new device auto-registers under its ID in Firebase

The dashboard auto-discovers all devices by listing `/devices/` in Firebase.
